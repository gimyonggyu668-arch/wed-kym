구문 분석
- 사용자가 작성한 SQL 문장을 DBMS가 이해할 수 있는 형태로 해석하는 과정 
SELECT * FROM EMP WHERE EMPNO = 7369;
DBMS는 이 문자열을 보고 
SELECT문인지 
문법이 올바른지
EMP 테이블이 존재하는지 
EMPNO 컬럼이 존재하는지
사용자가 권한이 있는지
같은 내용을 확인해야 한다.


구문 분석의 3단계
1. 문법 검사 : SQL문법이 맞는지 확인
2. 의미 검사 : 문법은 맞지만 실제 객체가 존재하는지 확인
3. 권한 검사 : 사용자가 해당 객체에 접근 가능한지 확인 

실행단계 
- 구문 분석이 끝난 SQL을 실제로 실행하는 단계 
- 구문 분석 단계에서 실행 계획이 만들어졌다면, 데이터베이스는 그 계획에 따라 실제 데이터를 찾거나 변경을 한다.

(예 : 개체 무결성, 참조 무결성 등)

제약조간

관계 데이터베이스 모델에서 상위 테이블의 기본키 (Primary Key)를 참조하여 테이블 간의 연결 고리 역할을 하는 속성을 무엇이라고 하는지 작성하시오.

논리 데이터 모델링의 주요 목적 중 하나는 데이터의 중복을 제거하여 이상현상
(Anomaly)을 방지하는 것입니다.
데이터베이스에서 발생할 수 있는 이상 현상 (Anomaly)의 3가지 종류를 제시하고, 각각의 개념을 간략히 설명하시오.


삽입이상
- 데이터를 추가할 때 필요없는 데이터도 같이 추가 
갱신이상
- 일부 정보 수정시 데이터의 불일치가 발생하는 현상 
데이터베이스 설계 과정에서 '개체 무결성 (Entity Integrity')과 '참조 무결성(Referential Integrity)'은 데이터의 정확성을 보장하는 핵심 규칙입니다.
이 두 제약조건의 개념과 위배되지 않기 위한 세부 규칙을 비교하여 서술하시오.

개체 무결성
- null 값이 들어가면 안되고, 중복되는 값이 들어가면 안된다.
참조 무결성





1정규형 (1NF) : 모든 테이블의 값이 원자값이야 한다.
2정규형 (2NF) : 1정규형을 만족하면서, 기본키가 아닌 모든 속성이 기본키에 완전 함수적종속을 만족해야 한다.
3정규형 (3NF) : 2차 정규형을 만족하고, 기본키가 아닌 모든 속성이 기본키에 이행함수적 종속을 만족하지 않아야 한다.


관계형 데이터 모델에서 릴레이션의 행 (Row)을 뜻하는 용어로, 카디널리티(Cardinality)의 수와 직결되는 이것은 무엇인가?

어떤 릴레이션 R에서 속성 X의 값 각각에 대해 속성 Y의 값이 오직 하나만 연관되어 있을 때, Y는 X에 '이것' 되어 있다고 한다. 무엇인가?

함수적 종속성

개념적 데이터 모델링 단계에서 도출된 ERD를 관계형 데이터베이스의 논리적 테이블 구조로 변환하는 과정을 '릴레이션 스키마 변환 (Mapping Rule)'이라고 합니다. 이 과정에서 1 : N (일대다) 관계를 가진 두 개체가 테이블로 변환될 때 외래키(Foreign Key)가 지정되는 규칙을 서술하시오.

파티셔닝

수평분할은 행 수가 너무 많아 성능이 저하되는 대용량 테이블 특정 컬럼 값의 범위를 기준으로 행 단위 분할 하는 것 

이로 인해 데이터를 검색할 때 전체 테이블을 뒤지지 않고 해당 조건이 속한 파티션 영역에만 집중 검색하므로 디스크 물리적 탐색 범위가 줄어들어 검색 속도가 획기적으로 상승 


반정규화를 수행하면 동일한 데이터가 여러 테이블에 중복저장되므로, 해당 데이터를 수정할 때 중복된 모든 곳을찾아 수정해야 하므로 성능 오버헤드가 증가한다.
만약 일부테이블만 반영되고 누락될 경우 데이터의 불일치가 발생하는 '데이터 무결성 파괴'및 이상현상이 발생한다.
 
집계 테이블 추가
대량의 과거 이력 데이터를 기반으로 정기적인 합계, 평균 등의 연산을 미리 수행하여 그 결과를 별도 테이블에 저장해 두는 방식

진행 테이블 추가 
여러 단계의 업무 프로세스 (예 : 주문 -> 결제 -> 배송)중 현재 진행중인 상태나 최신 트랜잭션 데이터만을 별도로 분리하여 빠른 조회가 가능하도록 관리하는 방식 

접근 속도를 높이기 위해 인덱스를 대량으로 추가하거나 반정규화를 통해 테이블 중복 저장을 하게 되면 데이터를 빠르게 찾을 수는 있지만 추가적인 물리 공간을 많이 소모하여 공간 효율이 떨어진다. 반대로 물리 공간을 최소화 하기 위해 극단적인 정규화를 수행하게 되면 공간 효율성은 극대화 되지만, 빈번한 조인 연산과 디스크 탐색으로 접근 속도는 느려진다.

데이터베이스의 성능을 최적화하기 위해 데이터가 저장되는 물리적 공간이자, 테이블과 인덱스가 실제로 할당되는 단위인 이것을 무엇이라 하는가?

데이터베이스 내에서 인덱스 (Index)를 무분별하게 많이 생성했을 때, 데이터 삽입 (Insert)

급여 테이블에서 주민번호나 계좌번호 같은 민감한 컬럼을 제외하고 '이름'과 '직급'만 조회하는 뷰를 생성해 일반 직원에게 권한을 부여함으로써 민감 정보 노출을 차단할 수 있다.

독립성 : 물리적 원본 테이블의 구조가 변경되더라도 외부 어플리케이션은 기존 구조에 맞춰 선언된 뷰를 계속 호출하므로, 어플리케이션 소스 코드를 수정하지 않아도 된다.

데이터베이스 오브젝트 중 '시퀀스(Sequence)'의 역할과 물리 데이터저장소 구성 시 왜 시퀀스를 사용하는지 그 필요성을 서술하시오.  

데이터베이스에서 고유한 순차적 정수 번호를 생성해주는 객체 물리 구성 시 동시 접속자가 많은 대규모 시스템에서 중복되지 않는 기본키 값을 얻기 위해 DBMS 자체에서 고속으로 독립적인 순번을 안전하게 발행하기 위해 사용한다.

반면 프로시저는 복잡한 로직과 여러 SQL을 DB서버 내에 하나의 프로그램으로 

트리거 (Trigger) 내부에서 무분별하게 DML 연산을 수행하거나 트랜잭션 제어어 (TCL)인 COMMIT 또는 ROLLBACK을 사용할 수 없는 물리적/논리적 이유를 서술하시오.
