[ Sub Query ] ===============================
- SQL문 안에 SQL문이 있는 것 
- 특정 조건을 다른 테이블 조회 결과로 판단할 때 
- 평균값, 최대값 같은 계산 결과를 기준으로 조회할 때
- Join 없이 단계적으로 데이터를 조회하고 싶을 때 

종류
- 단일 행 서브쿼리 : 결과가 1개
- 다중 행 서브쿼리 : 결과가 여러개 
 
select emp_name 
from employee 
where department = (
     select department 
     from employee
     where salary >= 6000
)
사용하는 기호 
IN : 여러 값 중 하나

- 다중 컬럼 서브쿼리 : 컬럼이 여러개

- from절 서브쿼리 : 가상 테이블 처럼 사용 

[ Join ] ====================================
- 정규화를 통해서 분리된 테이블에서 공통된 컬럼을 기준으로 연결해서
조회하는 기능

PK와 FK로 연결된 컬럼이 있어야 한다.

INNER JOIN
- 두 테이블에서 조건이 일치하는 데이터만 조회

LEFT OUTER JOIN 
- 왼쪽 테이블 (FROM 절에 작성된 테이블)의 데이터는 전부 조회하고, 오른쪽 
테이블(JOIN된 테이블)의 데이터는 조건이 맞는 데이터만 조회
- 없는 데이터 쪽은 NULL로 표시한다.

RIGHT OUTER JOIN 
- 오른쪽 테이블은 전부 조회하고, 왼쪽 테이블은 조건이 맞는 데이터만 조회
- LEFT와 RIGHT는 방향만 바뀐것이기 때문에 테이블 순서를 바꿔서 LEFT를 많이 사용한다.

CROSS JOIN 
- 두 테이블에 들어있는 데이터의 모든 조합을 만든다.

핵심문법 
SELECT 조회하고 싶은 컬럼
FROM 기준 테이블 별칭
JOIN 연결 테이블 별칭
ON 기준 테이블. 컬럼 = 연결 테이블.컬럼;

ON 절이 두 테이블을 연결하는 기준

외래키가 없어도 JOIN은 가능하나 무결성을 해칠 수 있기 때문에 권장하지 않는다.

무결성** 시험** LEFT**

[ View ] ===================================
- SELECT문의 결과를 가상의 테이블 처럼 저장해놓은 객체 
복잡한 SELECT문을 간다하게 하고 
VIEW를 만든다고 해서 실제로 데이터가 저장되는것은 아님
원본 데이터가 바뀌면 VIEW에 들어있는 데이터도 바뀐다.

VIEW안에서 VIEW를 사용 하는 것이 가능하기는 하나 성능 저하가 일어날 수 있다.


[ Index ] ===================================
- 조회를 빠르게 하기 위한 목적으로 사용
- where, order by, group by에 많이 사용되는 속성에 대해 미리 접근 위치를 기록해 놓는다.
- 실제 조회를 할 때 해당 컬럼이 사용이되면 index가 자동으로 사용 
- insert, update, delete에 대해서는 좀 불리하다.

index 생성
create index 인덱스명
on 테이블(컬럼)

테이블에 어떤 인덱스가 존재하는지 확인
SHOW INDEX FROM 테이블명;
AUTO_INCREMENT
- 숫자 값을 자동으로 1씩 증가시켜서 입력해주는 속성
- 데이터를 추가할 때 마다 직접 번호를 입력하면 번거롭기 때문에 사용을 한다.
- 반드시 숫자 타입의 컬럼에만 붙여야한다.
- 인덱스가 있어야 하기 때문에 PK와 함계 사용한다.
- 한 테이블에 하나만 사용할 수 있다. 
- 직접 값을 넣을 수도 있다. 
INSERT INTO BOARD(BOART_ID, TITLE, CONTENT)
VALUES (100, '관리자 공지', '직접 번호 입력');
그 다음 자동 증가 값은 보통 101부터 시작한다.
- 중간 번호는 재사용하지 않는다.
1     첫번쨰글
2    두번쨰글
3    세번째글

DELETE FROM BOARD
WHERE BOARD_ID = 3;

새 글을 추가하게 되면 BOARD_ID가 4로 추가된다.
- AUTO_INCREMENT는 연속 번호 보장용이 아니라 고유 식별자 생성용이다. 
- INSERT 했다가 ROLLBACK 해도 다음 번호로 건너뛴다.
- 테이블 생성 후 다음 자동 증가값을 바꿀 수 있다. 
ALTER TABLE 테이블명 AUTO_INCREMENT = 1000;
이후 새 데이터는 1000부터 들어갈 수 있다. 

- 현재 AUTO_INCREMENT 값 확인하기 
SHOW TABLE STATUS LIKE '테이블명';

- 마지막으로 생성된 번호 확인하기
SELECT LAST_INSERT_ID();

매개변수 IN, OUT, INOUT

out은 프로시저 내부에서 만든 값을 외부로 내보내는 매개변수
create procedure  프로시저명(ont 매개변수명 타입)

inout은 값을 받아오고, 수정한 뒤 다시 내보내는 역할
