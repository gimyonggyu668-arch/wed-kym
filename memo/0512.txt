MySQL의 예외처리
Handler
- Procedure 안에서 에러, 경고, 조회, 결과 없음 같은 상황을 처리하는 예외처리 문법
- 프로시저 실행중 문제가 생겼을 때 프로시저가 바로 중단되지 않도록 처리 방법을 정해두는 문법

에러 처리 : 중복된 PK를 INSERT한 경우 
조회 결과 없음 처리 : select ... into의 결과가 없는 경우 
커서 종료 처리 : 더 이상 읽을 데이터가 없는 경우 
ㄴ cursor : select 결과를 한 줄 씩 꺼내서 처리하는 기능
경고 처리 : 데이터 잘림 , 형 변환 경고 등

기본 문법
declare 핸들러타입 handler for 예외 
begin 
예외처리
end;

핸들러 타입 종류 
continue : 에러 처리 후 다음 문장 계속 실행 
exit : 에러 처리 후 begin - end 블록 종료

예외종류
sqlexception : 대부분의 SQL 에러
sqlwarning : SQL 경고
not found : 조회 결과 없음 
MySQL 에러 코드 : 1062 (중복키 ) 
AUTO_INCREMENT
- 숫자 값을 자동으로 1씩 증가시켜서 입력해주는 속성
- 데이터를 추가할 때 마다 직접 번호를 입력하면 번거롭기 때문에 사용을 한다.
- 반드시 숫자 타입의 컬럼에만 붙여야한다.
- 인덱스가 있어야 하기 때문에 PK와 함계 사용한다.
- 한 테이블에 하나만 사용할 수 있다. 
- 직접 값을 넣을 수도 있다. 
INSERT INTO BOARD(BOART_ID, TITLE, CONTENT)
VALUES (100, '관리자 공지', '직접 번호 입력');
그 다음 자동 증가 값은 보통 101부터 시작한다.
- 중간 번호는 재사용하지 않는다.
1     첫번쨰글
2    두번쨰글
3    세번째글

DELETE FROM BOARD
WHERE BOARD_ID = 3;

새 글을 추가하게 되면 BOARD_ID가 4로 추가된다.
- AUTO_INCREMENT는 연속 번호 보장용이 아니라 고유 식별자 생성용이다. 
- INSERT 했다가 ROLLBACK 해도 다음 번호로 건너뛴다.
- 테이블 생성 후 다음 자동 증가값을 바꿀 수 있다. 
ALTER TABLE 테이블명 AUTO_INCREMENT = 1000;
이후 새 데이터는 1000부터 들어갈 수 있다. 

- 현재 AUTO_INCREMENT 값 확인하기 
SHOW TABLE STATUS LIKE '테이블명';

- 마지막으로 생성된 번호 확인하기
SELECT LAST_INSERT_ID();

몇 개까지만 보여줘

FROM 테이블
LIMIT 개수

내림차순으로 정렬해서 상위 5개만 조회해라 

데이터가 몇백만건 들어있는 데 테이블에서 LIMIT이 없으면 위험할 수 있다.

My SQL에서 기본적으로 세미콜론을 만나면 SQL문장이 끝났다고 판단한다.

BEGIN ~ END는 여러 SQL문장을 하나의 블록으로 묶는 문법
BEGIN 
             문장1;
             문장2;
             문장3;
END

변수를 선언하는 DECLARE
DECLARE 변수명 자료형 DEFAULT 기본값;
DECLARE v_name varchar(50) default '홍길동'

BEGIN ~ END 블록의 가장 앞부분에 작성해야 한다.
일반 SQL문 뒤에 선언하면 오류가 난다.

BEGIN
END;

변수에 값을 넣는 SET
SET 변수명 = 값;


매개변수 IN, OUT, INOUT

IN은 외부에서 값을 받아오는 매개변수 
CREATE PROCEDURE 프로시저명(IN 매개변수명 타입)

out은 프로시저 내부에서 만든 값을 외부로 내보내는 매개변수
create procedure  프로시저명(ont 매개변수명 타입)

inout은 값을 받아오고, 수정한 뒤 다시 내보내는 역할
create procedure 프로시저명 (inout 매개변수명 타입)
