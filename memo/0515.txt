1. 현행 시스템의 분석 
- 새 시스템을 만들거나, 기존 시스템을 개선하기 위해 현재 사용중인 업무 방식, 프로그램 ,
데이터, 서버, 네트워크, 문제점을 조사하고 정리하는 활동 

목적
- 새로운 시스템을 정확하게 만들기 위해 현재 상황을 이해한다. 

하지 않을 때 문제점
문제점 
1. 기존 업무의 흐름을 모른다면 필요한 기능을 빠트릴 수 있다.
2. 기존 데이터 구조를 모른다면 데이터 이전 시 오류 발생 
3. 기존 시스템 연동을 모른다며 외부 시스템과 연결 실패 
4. 사용자 불편 사항을 모른다면 새 시스템을 만들어도 만족도가 낮다.
5. 현재 원인을 모른다면 문제 해결이 아닌 단순 재개발이 된다.

현행 시스템 파악 절차
1단계 : 현행  시스템의 구성, 기능, 인터페이스 현황 파악
- 현재 시스템이 무엇으로 구성되어 있고, 어떤 기능을 제공하고, 다른 시스템과 어떻게 연결되어 있는 지 파악

시스템 구성 현황 파악
- 현재 운영중인 시스템의 종류와 역할을 정리하는 것.
- 각 시스템이 어떤 업무를 담당하는지 까지 정리한다. 

기능 현황 파악
- 각 시스템이 제공하는 기능을 정리하는 것
 
인터페이스 현황 파악
- 시스템과 시스템이 데이터를 주고받는 연결 지점을 의미 
- ex 쇼핑몰
결제 api
택배사 api
문자 발송 api

2단계 : 현행 시스템의 아키텍처 및 소프트웨어 구성 현황 파악
- 현재 시스템이 어떤 구조로 만들어져 있고, 어떤 소프트웨어를 사용하고 있는지 파악

아키텍처 현황 파악
- 현재 시스템이 어떤 구조로 만들어져 있는 지 조사하는 일 

소프트웨어 구성 현황 파악 
- 현재 시스템이 어떤 소프트웨어를 사용하고 있는지 조사하는 일

2단계 산출물
- 시스템 아키텍처 구성도

문제 1. 운영체제의 역할 설명하기
운영체제(OS)의 역할은 사용자와 하드웨어 사이의 인터페이스 역할을 하며, 자원 관리(CPU, 메모리, 저장장치), 프로세스 관리, 파일 시스템 관리, 입출력 제어 등을 수행합니다.
운영체제가 없는 경우 발생하는 문제는 하드웨어를 직접 제어하는 하드코딩이 필요해집니다. 여러 프로그램을 동시에 실행할 수 없으며, 자원 충돌을 방지할 수 없어 시스템이 매우 불안정해집니다.



문제 2. 운영체제 종류 구분하기
Windows: 사용하기 쉬운 GUI(그래픽 사용자 인터페이스)를 제공하며, 범용성이 높고 다양한 소프트웨어 및 하드웨어 드라이버를 지원합니다.
Linux: 오픈 소스 기반으로 누구나 수정이 가능하며, 보안성이 높고 자원 효율이 좋아 서버 및 임베디드 환경에서 널리 쓰입니다.
UNIX: 대용량 시스템 및 메인프레임용으로 설계되어 높은 신뢰성과 성능을 자랑하며, 다중 사용자 및 다중 작업 처리에 최적화되어 있습니다.



문제 3. 신뢰도 요구사항 분석하기
온라인 쇼핑몰 서버는 24시간 운영되어야 한다. 운영체제 선정 시 신뢰도가 중요한 이유는 온라인 쇼핑몰은 24시간 중단 없는 서비스(High Availability)가 필수이기 때문입니다. 운영체제가 불안정하여 시스템이 다운될 경우, 매출 손실뿐만 아니라 사용자의 결제 정보 누락이나 신뢰도 하락으로 이어지기 때문에 장애 발생률이 낮고 복구 능력이 뛰어난 OS를 선정해야 합니다.



문제 4. 메모리 누수 설명하기
메모리 누수(Memory Leak)의 개념은 프로그램이 실행되면서 점유한 메모리를 사용 후 적절히 해제하지 않아, 사용 가능한 메모리 공간이 계속 줄어드는 현상입니다.
문제점은 시스템의 가용 메모리가 고갈되어 전체적인 성능 저하(Slowdown)가 발생하며, 결국 시스템이 비정상적으로 종료(Crash)될 수 있습니다.



문제 5. 동시 사용자 처리 분석하기
수강신청 시스템에서 오전 9시에 학생 5,000명이 동시에 접속하였다. 운영체제가 처리해야 하는 작업은 다음과 같습니다.
프로세스/스레드 생성 및 스케줄링: 각 사용자 요청을 처리할 작업 단위를 생성하고 CPU 시간을 공정하게 배분합니다.
메모리 관리: 수많은 세션 정보를 저장하기 위한 메모리 공간을 할당하고 보호합니다.
네트워크 I/O 제어: 대규모 패킷 데이터를 효율적으로 처리하여 병목 현상을 최소화합니다.



문제 6. 대용량 파일 처리 요구사항 분석하기
영상 업로드 서비스에서 운영체제의 파일 처리 성능이 중요한 이유는 영상이나 이미지 업로드 서비스는 디스크 쓰기 작업이 빈번하기 때문입니다. 운영체제의 파일 시스템 처리 성능이 낮으면 I/O 대기 시간이 길어져 사용자 체감 속도가 느려지고, 서버 전체의 처리량(Throughput)이 급격히 저하됩니다



문제 7. 32bit 운영체제와 64bit 운영체제의 차이점을 작성하시오.
차이점은 가장 큰 차이는 CPU가 한 번에 처리하는 데이터의 양과 메모리 인식 주소 범위입니다. 32bit는 최대 약 4GB의 RAM만 인식할 수 있습니다. 64bit는 16EB 이상의 방대한 RAM을 인식할 수 있어, 고성능 작업 및 대용량 데이터 처리에 필수적입니다



문제 8. 운영체제 선정 시 공급 벤더의 기술 지원이 중요한 이유를 작성하시오.
공급 벤더의 기술 지원이 중요한 이유는 기업용 시스템(특히 결제 시스템)에서 예상치 못한 보안 취약점이나 버그가 발견되었을 때, 신속한 패치 업데이트와 기술 자문을 받을 수 있어야 하기 때문입니다. 이는 서비스 중단 시간을 최소화하는 데 결정적인 역할을 합니다.



문제 9. 사용자가 많은 운영체제를 사용하는 장점을 작성하시오.
사용자가 많은 운영체제를 사용하는 장점은 사용자 층이 두터우면 커뮤니티와 정보 공유가 활발합니다. 문제 발생 시 해결 방법을 찾기 쉽고, 해당 OS를 다룰 수 있는 전문 인력을 구하기 용이하며, 서드파티 소프트웨어와의 호환성이 우수합니다.



문제 10. Linux가 서버 환경에서 많이 사용되는 이유를 작성하시오.
Linux가 서버 환경에서 많이 사용되는 이유는 오픈 소스이므로 라이선스 비용이 적거나 없으며(비용 절감), 시스템 구조가 투명하여 최적화가 쉽습니다. 또한, 텍스트 기반 제어(CLI)를 통해 원격 관리 및 자동화에 최적화되어 있고 보안성이 뛰어납니다



문제 11. 운영체제에서 드라이버가 필요한 이유를 작성하시오.
운영체제에서 드라이버가 필요한 이유는 운영체제가 마우스, 프린터, 그래픽 카드와 같은 다양한 하드웨어와 통신하기 위해서는 각 장치의 특성에 맞는 번역기가 필요하기 때문입니다. 드라이버가 있어야 OS가 하드웨어의 기능을 올바르게 제어할 수 있습니다.



문제 12. Linux 서버가 저사양 서버에서도 많이 사용되는 이유를 작성하시오.
Linux 서버가 저사양 서버에서도 많이 사용되는 이유는 Linux는 커널을 매우 가볍게 구성할 수 있고, GUI 없이 텍스트 모드(CLI)만으로 운영이 가능하기 때문입니다. 이 덕분에 불필요한 자원 낭비가 적어 낮은 CPU와 메모리 환경에서도 안정적으로 동작합니다.



문제 13. 다음 환경에서 비용 절감 효과가 큰 이유를 작성하시오.
비용 절감 효과가 큰 이유는 Windows나 UNIX와 달리 별도의 라이선스 구매 비용이 발생하지 않으며, 하드웨어 요구 사양이 낮아 고가의 고사양 장비를 도입하지 않아도 대규모 서버 팜을 구축할 수 있기 때문에 전체적인 TCO(총 소유 비용)가 크게 절감됩니다.
