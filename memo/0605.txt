UX(User eXperience) : 사용자 경험
- 사용자가 소프트웨어를 사용하며 느끼는 전체적인 경험을 의미한다.
- 예를 들어 웹을 사용하면서 느끼는 만족도, 편의성
- UX는 사용자가 소프트웨어를 통해 무엇을 얻고 느끼는지에 중점을 둔다.

UI/UX 구성요소
UI 구성 요소
- 시각적 디자인 : 색상, 타이포그래피, 레이아웃, 아이콘 등 사용자가 화면에서 보는 모든 요소 
- 상호작용 디자인 : 버튼 클릭, 스와이프 등 사용자의 조작에 대한 반응
- 정보 구조 : 콘텐츠의 구성 및 분류
UX 구성 요소
- 사용성 : 사용자가 소프트웨어를 얼마나 쉽게 이용할 수 있는지 정도
- 유용성 : 소프트웨어가 사용자의 요구사항을 얼마나 충족하는지 정도
- 매력도 : 소프트웨어가 사용자에게 얼마나 매력적인지 정도 
- 접근성 : 다양한 사용자가 소프트웨어를 얼마나 쉽게 이용할 수 있는지 정도
- 신뢰성 : 소프트웨어에 대한 사용자의 믿는 정도 

UI/UX의 관계
- 서로 밀접하게 관련이 있긴하다.
- UX를 위해서는 좋은 UI가 필수지만, 좋은 UI가 좋은 UX를 보장하지는 않는다. 
- 예를 들어 멋진 버튼이 있어도 사용하기가 불편하면 좋은 UX라고 할 수 없다.
- 사용하기 편리한 화면이라도 디자인이 매력적이 않다면 좋은 UX를 제공하기 어렵다 
- UI와 UX는 서로 보완하며 사용자의 만족도를 높이는 데 기여한다.

UI/UX 표준
- 과거에는 개발자나 기업의 관점에서 시스템을 설계하는 경우가 많았다.
- 하지만 최근에는 사용자의 편의성과 만족도가 중요해지면서 사용자 중심의 UI/UX 설계가 요구되고 있다.
- 개발자마다 서로 다른 방식으로 화면을 설계하면 사용자가 혼란을 느낄 수 있다.
- 시스템마다 UI가 다르면 매번 사용 방법을 새롭게 학습해야 한다. 
- 이러한 문제를 줄이기 위해 UI/UX 설계 시 공통적으로 참고할 수 있는 기준을 마련한것이 UI/UX 표준이다 

UI/UX 표준이 필요한 이유 
1. 일관성 있는 사용자 경험 제공
돋보기 아이콘 -> 검색
집 모양 아이콘 -> 홈
휴지통 아이콘 -> 삭제 
톱니바퀴 아이콘 -> 설정
- 공통된 설계 기준을 적용하면 사용자는 새로운 시스템도 쉽게 사용할 수 있다.

2. 사용성 향상
- 사용자는 시스템을 배우는 시간을 줄이고 원하는 작업을 더 빠르게 수행할 수 있다.
ex) 상품 검색 -> 상품 선택 -> 장바구니 -> 결제
- 이러한 표준적인 구조는 사용자가 쉽게 적응할 수 있도록 돕는다. 

3. 개발 품질 향상
- 표준을 적용하면 검증된 설계 원칙을 사용할 수 있으므로 품질이 향상된다.

4. 유지보수가 용이하다.
- 화면 구조와 설계 기준이 통일되어 있기 때문에 기능 추가와 수정이 쉬워진다.

5. 접근성 향상 
- 연령, 성별, 직무, 장애 여부와 상관없이 다양한 사용자가 시스템을 사용할 수 있도록 지원할 수 있다.

사용자 중심 설계의 6대 원칙
1. 사용자를 명확하게 이해하고 설계한다.
- 설계자는 사용자가 누구인지, 어떤 환경에서 시스템을 사용하는지, 어떤 목표를 가지고 있는지를 먼저 파악해야 한다.
예를 들어 은행 모바일앱을 설계한다면
사용자의 연령대
금융 서비스 이용 경험
스마트폰 활용 능력
사용환경 

2. 사용자가 설계 과정 전반에 참여시킨다.
- 사용자 중심 설계에서는 사용자가 단순한 평가 대상이 아니라 설계 과정의 중요한 참여자가 된다.
- 예를 들어 화면 시안을 제작한 후 실제 사용자에게 사용해보도록 하고 의견을 수집할 수 있다. 이를 통해 개발자나 디자이너가 미처 발견하지 못한 문제점을 확인할 수 있다. 

3. 사용자 평가를 기반으로 설계한다.
- 설계자는 사용자의 의견과 사용성 평가 결과를 바탕으로 지속적으로 시스템을 개선해야 한다.

4. 반복적인 설계 과정을 수행한다. 
- 사용자 중심 설계는 다음과 같은 반복과정을 거친다.
요구사항 분석 -> 프로토타입 제작 -> 사용자 평가 -> 개선 -> 재평가 

5. 사용자 경험 전체를 고려한다.
- 단순히 화면 디자인만을 의미하지는 않는다.
- 사용자가 서비스를 처음 접하는 순간부터 사용을 종료하는 시점까지의 모든 경험을 포함한다.

6. 다양한 전문가가 협력하여 설계한다.
- 효과적인 사용자 경험을 제공하기 위해서는 여러 분야의 전문가가 함께 참여해야 한다.

UI/UX 관련 키워드로 검색 
- 각 기관의 자료실 또는 정책 자료 게시판에서 UI/UX 관련 자료를 검색한다.
- 제공된 지침은 문서나 가이드라인 목록을 확인하고 최신 버전의 문서를 우선순위로 확인한다.
- 가이드라인은 시간이 지남에 따라 개정될 수 있다.
- 따라서 오래된 문서보다 최신 버전 문서를 우선적으로 검토해야 한다. 
- 관련 지침이나 가이드라인 문서를 다운로드받고 문서 개요 및 목차를 확인한다.

제공하는 지침이나 가이드라인의 접근성, 일관성, 색상 대비 등 핵심 항목을 발췌하여 정리한다.
- 자료의 주요 항목 선별 기준을 정의한다.
- 접근성, 일관성, 색상 대비 등 어떤 항목을 중심으로 발췌 할 것인지를 선정하고 불필요한 정보를 제외할 수 있도록 기준을 정의한다.
- 접근성과 관련된 항목을 정리한다.
- 장애인 접근성 보장을 위한 명시적 지침을 요약하여 정리한다. 
- 일관성과 관련된 항목을 정리한다. 
- 내비게이션 구성의 일관성, UI 패턴 동일성, 명칭 아이콘 버튼의 반복 사용 등에 관한 지침 내용을 확인하여 정리 
- 색상 대비와 관련한 항목을 정리한다.
- 최소 대비 비율과 같은 지침내용을 확인하여 정리한다.

공공기관 웹사이트 / 앱 사례에 적용된 가이드라인 요소를 분석한다.
- 공공기관 웹사이트 및 앱 분석 대상을 선정한다. 
- 웹사이트 또는 앱 중 하나 이상을 선정하고 공공기관인지 확인 후 사용 목적등을 파악하여 분석 대상을 선정한다. 
- 선정한 웹사이트 또는 앱에 적용된 가이드라인 요소 분석을 위한 항목을 정리한다.
- 접근성 표준 지침에서 선정한 웹사이트 또는 앱에 적용된 요소를 분석하기 위한 항목을 정리한다.
- 분석할 웹사이트 페이지, 기능 항목을 정리하고 가이드라인과 일치 여부 판단 기준을 설정한다.
- 각 항목에 대해 직접 사용한 후 기준에 따라 준수 여부를 체크하고 가이드라인 기준 미달 사례를 수집한다.
- 선정한 웹사이트에 대해 기준을 준수한 사례와 미흡한 사례를 분류하고 분석한다.
