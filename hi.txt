OSI 7계층

- 컴퓨터끼리 통신할 때 7단계로 나눠서 처리하는 구조

1계층 : 물리계층
- 데이터를 실제로 보내는 계층 
- 데이터의 상태가 bit (0과 1로 이루어짐) 이다.
- 전기, 빛, 전파와 같은 순수 신호 상태
- 안에 뭐가 들어있는지는 모름 
장비
- 허브 : 여러 컴퓨터를 연결하는 장치
- 리피터  : 약해진 네트워크 신호를 다시 강하게 만들어주는 장치 

2계층 : 데이터 링크 계층
- 같은 네트워크 안에서 데이터가 제대로 가도록 처리하는 계층
- 데이터 형태 : Frame
    ㄴ Packet에 MAC 주소와 오류 검출 정보가 붙은 것
- MAC 주소 : 인터넷을 연결할 수 있는 기기에 할당되어있는 고유한 ID
장비 
- 스위치 : 필요한 곳에만 데이터를 전송하는 장비 
    ㄴ 스위치가 어느 장비로 데이터를 전송할지 판단할 때 사용하는 것이 MAC 주소다.
- 브리지 


3단계 : 네트워크 계층
- 서로 다른 네트워크를 넘어 목적지까지 가는 길을 찾는 계층
- IP 주소를 사용한다.
데이터 상태 : Packet
ㄴ Segment에 출발지와 목적지의 IP주소 정보가 붙은 상태 
장비
라우터 : 서로 다른 네트워크를 연결해주는 장비 
L3 스위치 : 스위치의 기능에 라우터 기능을 합친 장비 



4계층 전송계층(Transport Layer)
- 데이터를 정확하고 안정적으로 전달할지, 빠르게 전달할지 결정하는 계층
계층
데이터 상태: Segment (상위계층에서 내여론 Data에 프로세스 간 통신을 위한 정보가 붙은 상태)
장비 
L4 스위치 : IP + 포트번호를 기준으로 트래픽을 분해하는 장비 
로드 밸런서 : 서버 여러 대에 트래픽을 균등하게 나눠주는 장비 
일부 방화벽

5계층 : 세션 계층
- 통신의 시작, 유지, 종료를 관리하는 계층
데이터의 상태 : Data

6계층 : 표현 계층
-데이터의 표현 방식을 맞춰주는 계층
- 데이터를 보기 좋게 변환단다.
데이터 상태
Data

7계층 : 응용 계층
- 사용자가 직접 만나는 계층
- 웹 브라우저, 메이 프로그램, 파일 전송 프로그램 들이 관련이있다.
데이터 상태
- Data : 여기서는 아직 사람이 이해할 수 있는 실제 내용
- 하드웨어 장비보다는 소프트웨어 서비스와 더 관련이 깊다.

데이터 전송 흐름 
7 -> 6 -> 5 -> 4 -> 3 -> 2 -> 1
1.응용 -> HTTP요청을 생성
2. 표현 -> 암호화 / 변환
3. 세션 -> 연결 생성
4. 전송 -> TCP + 포트
5. 네트워크 -> IP주소 
6. 데이터링크 -> MAC주소
7. 물리 -> 신호로 전송


1 -> 2-> 3 -> 4-> 5 -> 6 -> 7
1. 신호 수신
2. 프레임 확인
3. IP확인
4. 포트 확인
5. 세션 확인
6. 데이터 해석
7. 프로그램 전달
