# interface  이번 주 수업에서 정리 못한 개념 

##  우선 인터페이스는 클래스가 아니다.
- 헷갈리는 포인트 인터페이스는 상속이 아니고 구현이다.
- 인터페이스는 여러 기능으로 나눌 수 있다.
- 여러 개를 구현하고 나중에 교체를 할 수 있다는 장점이 있다.


## interface의 중첩
 
 - 일단 클래스 안에 인터페이스를 정의 후에 "이 인터페이스는 여기 클래스랑 관련이 있어" 라고 묶는 느낌이다.
 - 이유 : 그냥 인터페이스를 만들면 밖에서 아무나 쓸 수 있기 때문에 
 ### 클래스 안에 인터페이스를 쓸 때는 "클래스명.인터페이스이름" 이렇게 쓴다.
