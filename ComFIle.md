# Java 작동 원리 시스템 

```
Main.java
   ↓ (컴파일: javac)
[바이트코드 생성]
Main.class
   ↓ (JVM 시작)
[클래스 로딩]
   ↓
[실행 엔진 동작]
   ↓
[실제 실행 (런타임)]
   ↓
프로그램 종료
```

### 컴파일
컴파일을 실행하기 전에 화면에 쓰느 코드를 읽고
- IDE가 먼저 검사

실행 버튼 클릭 

- 컴파일러  (javac)

검사 후 결과를 -> 바이트 코드를 통해 전달

JVM 로딩 

- 가상의 컴퓨터로 안에 코드의 안정성, 메모리 침범, 이상한 바이트 코드가 아닌지 확인한다.

Interpreter(인터프리터)

a = b +c;
- 한 줄씩 읽고 바로 실행 

JIT

- 자주 쓰는 코드를 발견하면 기계어로 변환 
처음에는 안쓰고 반복되면 사용 
이후 속도 향상됨
