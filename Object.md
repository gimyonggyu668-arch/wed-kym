# 설계도(class) 만들기

## 필드(Field): 객체의 정보 (데이터를 담는 변수) 
나이, 이름, 잔액 등을 담는 변수

## 1. 클래스를 불러오는 역할: new + 생성자
우리가 클래스라는 설계도를 가지고 실제 물건(객체)을 만드는 과정을 **"인스턴스화"**라고 하죠. 이때 사용하는 게 메서드가 아니라 생성자입니다.

코드: Student s1 = new Student();

역할: "Student라는 설계도를 불러와서, 메모리라는 공장에 실제 학생(s1) 한 명을 만들어줘!"라는 뜻입니다. 이때 Student() 부분이 바로 생성자입니다.

## 2. 메서드(Method)의 진짜 역할: "동작"
객체가 만들어진 다음에, 그 객체한테 **"야, 너 이거 해봐!"**라고 시키는 기능입니다.

코드: s1.study();, s1.eat();

역할: 이미 태어난 학생(s1)에게 '공부하기'나 '먹기'라는 구체적인 행동을 시키는 겁니다.

-----------------------------

### 1-1.클래스를 "불러와서 만드는" 단계 (New)
실행하는 방(Main)에서 가장 먼저 해야 하는 일은 설계도(클래스)를 보고 실물을 만드는 겁니다.

코드: Student s1 = new Student();

의미: "Student 설계도를 가져와서(불러와서) 메모리에 s1이라는 실물을 만들어라!"

역할: 이때 생성자가 작동하면서 객체가 탄생합니다. (이게 진짜 '불러오기'에 가깝습니다.)

2-1. 메서드를 "호출해서 일을 시키는" 단계 (Call)
객체가 만들어진 후에, 우리가 실행방에서 메서드 이름을 쓰는 건 **'명령'**을 내리는 겁니다.

코드: s1.setAge(20);

의미: "이미 만들어진 s1아, 너한테 있는 setAge라는 **기능(메서드)**을 지금 실행해!"

역할: 메서드는 클래스 전체를 불러오는 게 아니라, 그 안에 정의된 **'특정 동작 한 줄기'**를 실행하는 겁니다.
